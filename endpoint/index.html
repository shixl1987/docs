    OpenAPI Endpoint Description

    · queryAll

    Retrieve a complete list of Sengled devices and their parameter values belonging to the authenticated Sengled user account.

    GET https://base_url/openapi/api/v1/queryAll

    Input parameters

    None

    Response

    If success, a list of devices and their parameter values will be returned as a JSON device data object mentioned above. If failed, a JSON status code will issued as follows.

    Example: {“code":”100”,"message":" Invalid or missing required parameter."}

    Parameter name Type Description
    code String See Status code list for detailed code
    message String Detailed status message

    · query

    Query for device parameter values of a specified set of devices.

    POST https://base_url/openapi/api/v1/query

    Input parameters

    A list of device’s product type and MAC address.

    {

    ​ "devices": [

    ​ {

    ​ " product ": "element",

    ​ " deviceID ": " B0:CE:18:08:00:01"

    ​ }

    ​ ]

    }

    Name Description Type Required
    deviceID Device unique identifier String Yes
    product Product type String Yes

    Response

    If success, a list of devices and their parameter values will be returned as a JSON device data object mentioned above. If failed, a JSON status code will be issued as follows.

    Example: {“code":”100”,"message":" Invalid or missing required parameter."}

    Parameter name Type Description
    code String See Status code list for detailed code
    message String Detailed status message

    · control

    Adjust specified parameter settings for the specified device.

    POST https://base_url/openapi/api/v1/control

    Input parameters

    {

    ​ "product": "element",

    ​ "deviceID": "deviceID",

    ​ "propertyList": [

    ​ {

    ​ "property": "brightness",

    ​ "value": "0-100"

    ​ }

    ​ ]

    }

    Supported property names：

    Name Description Supported product Required Data type Valid values
    deviceID Device Unique ID ALL Yes String MAC address
    Product Product Type All Yes String snap，element, …
    switch Switch On/off state ALL No String off/on
    brightness Dim level ALL No String 0-100
    colorTemperature Color temperature Element No String 0-100
    microphone Microphone state Snap No String off/on
    motionDetection Is motion detection enabled? Snap No String off/on
    notification Is motion notification enabled? Snap No String off/on
    nightVision Is night vision enabled? Snap No String off/auto
    resolution Set camera resolution Snap No String 360/720/1080
    deviceReboot Reboot firmware? Snap No String reboot
    video Is video streaming enabled? Snap No String off/on
    snapshot Take a snapshot? Snap No String true
    audioOut Start/stop audio from the cloud to the Snap.(This needs to be used together with getSpeakerURL Snap No String start/stop

    Response status

    If all the parameters are adjusted correctly, a success status code will be issued as follows.

    {“code":”200”,"message":"success"}

    Otherwise, an example of the following error code will be returned.

    {“code":”100”,"message":" Invalid or missing required parameter."}

    Parameter name Type Description
    code String See Status code list for detailed code
    message String Detailed status message

    · getSpeakerURL

    This endpoint allows third-party service to acquire the RTSP link for playing audio into the Snap speaker.

    Note:

    After such a RTSP connection is connected, please invoke the openAPI endpoint device: audioOut to start and stop audio streaming into the Snap speaker.

    POST https://base_url/openapi/att/api/v1/getSpeakerURL

    Input parameters

    {

    ​ "product": "snap",

    ​ "deviceID": "deviceID"

    }

    Type Description Supported product Required Data type Valid values
    deviceID Device unique identifier Snap Yes String MAC address
    product Device product Snap Yes String snap，element, …

    Response

    A URL containing audio broadcasting address will be provided if successful.

    {

    ​ "voiceUrl": "rtsps://xxx.xxx.xxx.xxx:port/140A4524FC57C21DA0768EBEAB4AA392.sdp"

    }

    Name Type Supported product Description
    voiceUrl String Snap RTSPS audio broadcasting URL

    Otherwise, an error status code will be issued:

    {“errorCode":”20”,"message":" Invalid or missing required parameter."}

    Parameter name Type Description
    errorCode String See Status code list for detailed code
    errorMsg String Detailed status message
