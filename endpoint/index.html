    OpenAPI Endpoint Description

    ·queryAll

    Retrieve a complete list of Sengled devices and their parameter values belonging to the authenticated Sengled user account.

    GET https://openapi-router.cloud.sengled.com/openapi/api/v1/queryAll

    Input parameters

    None

    Response

    If success, a list of devices and their parameter values will be returned as a JSON device data object mentioned above. If failed, a JSON status code will issued as follows.

    Example: {“code":”100”,"message":" Invalid or missing required parameter."}

    Parameter name Type Description
    code String See Status code list for detailed code
    message String Detailed status message

    ·query

    Query for device parameter values of a specified set of devices.

    POST https://openapi-router.cloud.sengled.com/openapi/api/v1/query

    Input parameters

    A list of device’s product type and MAC address.

    {

    ​ "devices": [

    ​ {

    ​ " product ": "element",

    ​ " deviceID ": " B0:CE:18:08:00:01"

    ​ }

    ​ ]

    }

    Name Description Type Required
    deviceID Device unique identifier String Yes
    product Product type String Yes

    Response

    If success, a list of devices and their parameter values will be returned as a JSON device data object mentioned above. If failed, a JSON status code will be issued as follows.

    Example: {“code":”100”,"message":" Invalid or missing required parameter."}

    Parameter name Type Description
    code String See Status code list for detailed code
    message String Detailed status message

    ·control

    Adjust specified parameter settings for the specified device.

    POST https://openapi-router.cloud.sengled.com/openapi/api/v1/control

    Input parameters

    {

    ​ "product": "element",

    ​ "deviceID": "deviceID",

    ​ "propertyList": [

    ​ {

    ​ "property": "brightness",

    ​ "value": "0-100"

    ​ }

    ​ ]

    }

    Supported property names：

    Name Description Supported product Required Data type Valid values
    deviceID Device Unique ID ALL Yes String MAC address
    Product Product Type All Yes String element, …
    switch Switch On/off state ALL No String off/on
    brightness Dim level ALL No String 0-100
    colorTemperature Color temperature Element No String 0-100
    volume The volume of speaker Flex No String 0-100
    speaker The state of speaker:play,pause,stop Flex No String start/pause/stop
    rgbColorR Rgb red element No String 0-255
    rgbColorG Rgb green Element No String 0-255
    rgbColorB Rgb blue Element No String 0-255

    Response status

    If all the parameters are adjusted correctly, a success status code will be issued as follows.

    {“code":”200”,"message":"success"}

    Otherwise, an example of the following error code will be returned.

    {“code":”100”,"message":" Invalid or missing required parameter."}

    Parameter name Type Description
    code String See Status code list for detailed code
    message String Detailed status message

    ·register

    Allows third party services to register for callback service that provides asynchronous device parameter changes and/or events.

    POST https://openapi-router.cloud.sengled.com/openapi/api/v1/register

    Input parameters

    {

    ​ "registerUrl": " https://callback_url",

    ​ "username": "callback username",

    ​ "password": "callback password"

    }

    Name Description Required Data type
    registerUrl Callback URL Yes String
    username Callback username No String
    password Callback password No String

    Response status

    If registration is successful, a success status code will be issued as follows.

    {“code":”200”,"message":"success"}

    Otherwise, an example of the following error code will be returned.

    {“code":”100”,"message":" Invalid or missing required parameter."}

    Parameter name Type Description
    code String See Status code list for detailed code
    message String Detailed status message
