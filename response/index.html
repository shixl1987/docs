    RESPONSE

    Response status Code

    Responses from Sengled cloud will include HTTP status code and OpenAPI status code. OpenAPI status code supplements HTTP status code by providing more status details and/or more targeted error messages associated with specific product type.

    HTTP Status code

    HTTP Code Description
    200 OK
    400 Bad Request. The request is not properly formatted.
    401 Unauthorized. API authentication failed.
    403 Forbidden, No valid access-token on request.
    404 The server has not found anything matching the Request-URI.
    405 A request was made of a resource using a request method not supported by that resource (e.g.,using PUT on a REST resource that only supports POST).

    Sengled Open API Status Code

    Below is status code value description.

    200: Success

    <200: General errors

    >200: Error codes for Specific Sengled product module

    Return status data format：

    {"code":200,"message":"success"}

    General OpenAPI code list：

    Code Description
    200 OK
    100 Invalid or missing required parameter.
    101 Operation failed.
    102 Operation not supported.
    103 No Permission
    104 Invalid or expired access token
    105 One or more device(s) doesn’t exist.
    106 Other

    Openapi error code list specific for Sengled Element module：

    Code Description
    20602 hub offline
    20604 Sengled bulb does not exist
    20608 Bulb configuration failure
    20610 is not paid service user
    20007 Bulb offline

    Device data object

    The profile and settings of a Sengled smart device is represented as a data structure using the standard JSON format as follows.

    {

    "deviceList": [ //bulb list

    ​ {

    ​ "propertyList": [

    ​ {

    ​ "property": "name",

    ​ "value": "ElementBulb007E" //bulb name

    ​ },

    ​ {

    ​ "property": "brightness",

    ​ "value": "47" //bulb brightness:range(0-100)

    ​ },

    ​ {

    ​ "property": "switch",

    ​ "value": "on/off" //bulb power status

    ​ },

    ​ {

    ​ "property": "isOnline",

    ​ "value": "true" //bulb online status

    ​ },

    ​ {

    ​ "property": "startTime", //device activation time.

    ​ "value": "2015-11-05 08:15:30"

    ​ }

    ​ ],

    ​ "product": "element", //product type

    ​ "deviceType": "",

    ​ "deviceID": "B0CE18140000007E", //device MAC address

    ​ }

    ​ ]

    ​ }

    Device data object parameters:

    Name Type Description
    deviceList List List of device data object
    Property/value {String/String} Parameter name and value
    product String Product type（element…）
    deviceID String Device’s unique identifier

    propertyList values：

    Name Type Supported product type Description
    name String All Device name
    brightness String All Dim level（0-100）
    colorTemperature String Element Color temperature（0-100）
    Switch String All Light switch status on/off
    Power String Element Wattage
    isOnline String All Online status: true/false
    startTime String All UTC time at which a device is first activated.
    parentDeviceID String Element Hub id
    deviceType String Element Sub type of element
    volume String Flex The volume of speaker
