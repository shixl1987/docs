    Overview

    This document defines a set of cloud APIs that can be used by third-party services to access and control Sengled smart lighting devices via cloud-to-cloud integration. Access to the API endpoints requires a third-party service to authenticate itself using the standard OAuth2.0 authentication mechanism. Once a third-party service has successfully obtained an access token, it will authenticate all calls to any API endpoint by including the encoded access token in the Authorization header.

    Please see the detail of OAuth2 process in the diagram below.

    Here is the URL form used in Step 3:

    https://openapi.cloud.sengled.com/oauth2/authorize?response_type=code&client_id=testclient&redirect_uri= thirdpartner_url &scope=read&state=example

    The response returned in Step 4 includes:

    http://thirdpartner_url?code=b8ddea14-bad3-4181-b978-85c8ab9b9f44
